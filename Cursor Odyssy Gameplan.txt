# 테마: 태그 기능 가지 크레이트터 기능

- 회피가 기능
- 성장 목표: 조직 / 재료 / 스킬 / 패시ube 획득
- 로그라이크 형식 (가장 기본적인 리턴 시스템)

---

# 1. 기본 능력치 구조

| 기본 능력 | 설명 |
|:---|:---|
| 체력 | 전투 계약 |
| 공격력 | 기능 개다, 트랙 모드|
| 방어력 | 보조, 방어 가능 |
| 이동속도 | 방향 전투 가능 |
| 가장 | 기능 실험 가능 |
| 치유력 | 체력 회복 |

---

# 2. 스킬 / 패시ube 설계

- 스킬 20종
- 패시ube 3종으로 지정
- 한 회에 4가지 스킬, 5가지 패시ube 설정 가능
- 스킬 가지수 제한 구조

### 스킬 예제
- 파이어볼: 화염 포탈, 구매 후 적에게 피해
- 아이스월: 가르내는 방어포 처리
- 데시: 가비 규비조정 이동

### 패시ube 예제
- 화염증폭: 화염계 당 피해 증가
- 스트리스트: 시전 최규 도적 향상


---

# 3. 스킬 강화 / 진화 시스템

- 강화 포인트 획득
- 스킬 시간 감소, 포츠 증가
- 패시ube 개별 진화 가능 (ex. 파이어볼 + 모든 화염계 후경 가장 강화)


---

# 4. 맵 구조

- 맵 배치: 일정 가정의 규별 형식
- 슬보터 기능 발동
- 모니스터 스포인: 가까워지는 구조, 현재 플레이어와 300m 이상 거리 유지


---

# 5. 메치지 시스템

- 대기여일 시작 (10초 수)
- 10인 이상을 넘어야 시작
- 게임중 상입 가능 (300m 거리, 무적 5초, 레벨 고정)
- 공간 무료 재입


---

# 6. 랭킹 시스템

- 랭킹 기준: KDA (사망 / 파기 / 사용), 사용 시간, 모니스터 처치 횟수
- 결과를 계산해 여론 랭킹 순서 정해
- 랭킹 등경은 후에 게임 게임 결과 표시 계획


---

# 7. 진행 계획

- 현재 다 개발자: Cursor AI (프로머프트 가상)
- 처음은 기능 구조 / 구현 조합 건설 기준
- 후에 결제 / 건물 / 가계 가치 / 개인 결제 시스템 구조 참가 계획
